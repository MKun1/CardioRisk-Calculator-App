02 — Features

What drives predictions

Feature importance scores from the Decision Tree show how much each variable contributes to reducing prediction uncertainty across all 630,000 training samples. Values sum to 1.0.

Strongest protective factor
Thallium — Normal (3)
Patients with a normal thallium result are the first branch in the Decision Tree — immediately routed low-risk before any other feature is considered.
Strongest risk indicator
Chest pain type 4 — Asymptomatic
Counter-intuitively, no chest pain is the highest-risk type. 69.7% of asymptomatic patients in training data had disease.
Near-zero predictors
Blood pressure & fasting blood sugar
BP had a correlation of −0.01 with the target. Both features are retained but contribute negligible signal in this dataset.

04 — Pipeline

How the model was built

The full ML pipeline — from raw CSV to live prediction — in eight stages.

01
Data loading & inspection
630k rows loaded. Zero missing values confirmed. Target column decoded from "Presence"/"Absence" strings to binary 0/1.
02
Exploratory data analysis
Distribution plots, class-stratified histograms, categorical disease-rate tables, and a Pearson correlation matrix across all 13 features.
03
Preprocessing
Three nominal columns one-hot encoded (10 dummy columns). Five continuous columns StandardScaled. Scaler fitted on train only — applied identically to test.
04
Model training
5-fold stratified cross-validation. Decision Tree: max_depth=10, min_samples_leaf=50. Logistic Regression: saga solver, 1000 iterations, C=1.0.
05
Evaluation
Accuracy, F1, precision, recall, ROC-AUC across all folds. Confusion matrices on a 20% held-out validation slice (126,000 rows).
06
Feature importance
Gini reduction scores from the Decision Tree. Absolute coefficients from Logistic Regression. Rank agreement table comparing both models.
07
Model serialisation
Decision Tree, StandardScaler, feature column list, and nominal column list saved as .pkl files via Python's pickle module.
08
Web deployment
Flask backend loads artefacts once at startup. Incoming form data preprocessed through the identical pipeline before prediction.
05 — Limitations

What this tool cannot do

Not a medical diagnosis
This tool produces a statistical risk estimate — not a clinical diagnosis. A result of "High Risk" does not mean you have heart disease. Only a qualified clinician can diagnose.
Synthetic training data
The dataset is synthetically augmented from the UCI Heart Disease repository (~300 original records). While metrics are stable, real-world generalisation may differ.
No hyperparameter tuning
Default regularisation and depth settings were used. A properly tuned Random Forest or XGBoost model would likely outperform this Decision Tree.
No probability calibration
The raw probability scores from the Decision Tree are not calibrated. For clinical use, Platt scaling or isotonic regression would be required to make probabilities trustworthy.
